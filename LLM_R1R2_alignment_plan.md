# LLM R1/R2 Alignment Instance Plan

## Goal

The goal is to formalize a Lean instance showing that an observation-only
alignment check can leave a residual distinction between causal trajectories,
and that closing this residual requires a finite mediator of exact dimension.

The intended theorem is not a claim about real deployed LLMs yet.  It is a
formal structural theorem about an abstract model of terminal observation,
latent trajectory, and mediated alignment certification.

The target statement is:

```text
For every n >= 2, there is an abstract alignment carrier whose terminal R1
observation identifies n causal trajectories, while the R2 target distinguishes
them.  A mediator M : S -> Fin n closes the residual, every smaller mediator
S -> Fin m with m < n fails, and the mediator does not descend to the visible
marginal interfaces taken in isolation.
```

## Core Interpretation

The abstract carrier should represent full causal states, not only terminal
outputs.

```text
S = complete trajectory states
R1 = visible terminal observation
R2 = latent trajectory / causal decision coordinate
M  = causal certificate of that coordinate
```

The formal reading:

```text
R1 says:
  these states are observationally identical at the terminal interface.

R2 says:
  these states must still be distinguished because their causal trajectories
  differ.

M says:
  here is the finite mediator that restores exactly the missing trajectory
  coordinate.
```

## Minimal Abstract Model

The first target should be a finite abstract model, not an empirical neural
network model.

For every `n >= 2`, define a carrier indexed by `Fin n`.

```lean
inductive AlignmentState (n : Nat)
  | trajectoryClass : Fin n -> AlignmentState n
```

This is intentionally small.  The nontrivial content comes from the R1/R2
interfaces and exact dimension proof, not from adding ad hoc fields.

This inductive form is preferred over a free structure field because the state
is generated by the causal trajectory constructor itself.

If a richer carrier is wanted later, it must be dependent on the trajectory
constructor, not a free product of visible fields and a coordinate.

Allowed pattern:

```lean
inductive AlignmentState (n : Nat)
  | trajectoryClass :
      (i : Fin n) ->
      PromptClassFor i ->
      TerminalOutputFor i ->
      DecisionTraceFor i ->
      AlignmentState n
```

Avoid a free record of unrelated fields.

The first proof should keep only what is needed to certify the invariant.

## No Artificial Label Discipline

The Lean carrier must not attach an arbitrary certificate coordinate to an
otherwise independent visible state.

Avoid:

```lean
structure BadAlignmentState (n : Nat) where
  terminalOutput : TerminalOutput
  coordinate : Fin n
```

unless the coordinate is explicitly part of the modeled causal trajectory.

The clean minimal version is:

```lean
inductive AlignmentState (n : Nat)
  | trajectoryClass : Fin n -> AlignmentState n
```

and the visible observations are defined from the state without reading
the trajectory index.

If a richer model is introduced, the trajectory/certificate coordinate must be
determined by the causal constructor, not freely paired with an arbitrary
terminal observation.  The pattern should follow the corrected ZFC design:

```lean
inductive AlignmentState (n : Nat)
  | trajectoryClass : Fin n -> AlignmentState n
```

or a dependent structure where the proof fields tie the visible state to the
trajectory-generation rule.

The audit question is:

```text
Can two states with the same visible R1 observations have different R2
trajectory coordinates because they are different constructor-generated causal
trajectories, rather than because an external label was attached afterward?
```

The answer must be yes.

## Interfaces

The interface must be split into visible marginal readers, as in the refined
ZFC file.

Recommended interface:

```lean
inductive AlignmentInterface
  | terminalOutputReader
  | promptClassReader
```

For the first minimal model, both readers may be constant.

```lean
inductive AlignmentObservation
  | terminalOutput : TerminalOutput -> AlignmentObservation
  | promptClass : PromptClass -> AlignmentObservation
```

The active family:

```lean
def I_alignment : Subfamily AlignmentInterface
  | terminalOutputReader => True
  | promptClassReader => True
```

Marginal subfamilies:

```lean
def I_alignment_terminal_only : Subfamily AlignmentInterface
def I_alignment_prompt_only : Subfamily AlignmentInterface
```

The proof must include explicit non-descent to both marginal subfamilies.

The file should also prove:

```lean
terminal_only_proper_alignment :
  Subfamily.Proper I_alignment_terminal_only I_alignment

prompt_only_proper_alignment :
  Subfamily.Proper I_alignment_prompt_only I_alignment
```

These lemmas prevent the marginal claims from being only informal.

## R1 Observation

The R1 observation must not read the trajectory coordinate.

For the minimal model:

```lean
inductive TerminalOutput
  | sameVisibleOutput
deriving DecidableEq

inductive PromptClass
  | samePromptClass
deriving DecidableEq
```

Then:

```lean
def obs_alignment {n : Nat} :
    AlignmentInterface -> AlignmentState n -> AlignmentObservation
  | AlignmentInterface.terminalOutputReader, _ =>
      AlignmentObservation.terminalOutput TerminalOutput.sameVisibleOutput
  | AlignmentInterface.promptClassReader, _ =>
      AlignmentObservation.promptClass PromptClass.samePromptClass
```

This encodes the core phenomenon:

```text
all trajectories look identical to R1.
```

The file should expose this as a lemma:

```lean
jointSame_alignmentStates :
  ∀ i j : Fin n,
    JointSame obs_alignment I_alignment
      (AlignmentState.trajectoryClass i)
      (AlignmentState.trajectoryClass j)
```

This lemma is the exact formal content of terminal observational collapse.

## R2 Target

The R2 target reads the causal trajectory coordinate:

```lean
def sigma_alignment {n : Nat} (s : AlignmentState n) : Fin n :=
  match s with
  | AlignmentState.trajectoryClass i => i
```

This means:

```text
R2 requires trajectory-level distinction.
```

## Mediator

The mediator is the causal certificate:

```lean
def M_alignment {n : Nat} (s : AlignmentState n) : Fin n :=
  match s with
  | AlignmentState.trajectoryClass i => i
```

This is acceptable only because the carrier is explicitly a trajectory state.
The coordinate is the modeled causal trajectory itself, introduced by the
constructor of `AlignmentState`.

Audit rule:

```text
M must be definitionally recovered from the trajectory constructor.
It must not be a separate coordinate attached after terminal observations are
chosen.
```

## Canonical Pair

For `h : 1 < n`, define:

```lean
def firstTrajectory {n : Nat} (h : 1 < n) : Fin n := ...
def secondTrajectory {n : Nat} (h : 1 < n) : Fin n := ...

def x_alignment {n : Nat} (h : 1 < n) : AlignmentState n :=
  AlignmentState.trajectoryClass (firstTrajectory h)

def y_alignment {n : Nat} (h : 1 < n) : AlignmentState n :=
  AlignmentState.trajectoryClass (secondTrajectory h)
```

Then prove:

```lean
requiredAtCanonicalPair_alignment :
  RequiredDistinction sigma_alignment x_alignment y_alignment

jointSameAtCanonicalPair_alignment :
  JointSame obs_alignment I_alignment x_alignment y_alignment

canonicalDiagonalWitness_alignment :
  DiagonalizationWitness obs_alignment sigma_alignment I_alignment
    x_alignment y_alignment
```

## Mediated Closure

Prove:

```lean
mediatedResidualEmpty_M_alignment :
  MediatedResidualEmpty obs_alignment sigma_alignment I_alignment M_alignment
```

Proof idea:

```text
If sigma x != sigma y, then M x != M y because both are the trajectory
coordinate.
```

## Marginal Irreducibility

For every proper active subfamily:

```lean
witnessedIrreducibleMediator_M_alignment :
  WitnessedIrreducibleMediator obs_alignment I_alignment M_alignment
```

Then derive:

```lean
irreducibleMediator_M_alignment :
  IrreducibleMediator obs_alignment I_alignment M_alignment
```

Also expose explicit marginal non-descent:

```lean
no_descent_terminal_only_alignment :
  ¬ MediatorDescendsSubfamily obs_alignment I_alignment_terminal_only M_alignment

no_descent_prompt_only_alignment :
  ¬ MediatorDescendsSubfamily obs_alignment I_alignment_prompt_only M_alignment
```

These are important because they prove that no isolated visible marginal
recovers the causal trajectory coordinate.

The proof should not rely only on the abstract statement
`IrreducibleMediator`.  It should also expose direct witnessed statements:

```lean
witness_no_descent_terminal_only_alignment :
  MediatorNonDescentWitness
    obs_alignment I_alignment_terminal_only M_alignment

witness_no_descent_prompt_only_alignment :
  MediatorNonDescentWitness
    obs_alignment I_alignment_prompt_only M_alignment
```

This makes the terminal-only and prompt-only failures auditable as concrete
state pairs.

## Exact Dimension `n`

The lower bound should follow the same pattern as ZFC.

Prove that any mediated closure:

```lean
M : AlignmentState n -> Fin m
```

induces an injection:

```lean
Fin n -> Fin m
```

by:

```lean
fun i => M (AlignmentState.trajectoryClass i)
```

The injection proof must use the fact that for all `i j : Fin n`, the states
`trajectoryClass i` and `trajectoryClass j` are R1-indistinguishable.  If a
candidate mediator identifies them, mediated closure would contradict the R2
distinction `i != j`.

Then use:

```lean
FiniteDimensionHierarchy.no_injective_fin_of_lt
```

to prove:

```lean
no_smaller_mediatedR2Certificate_alignment :
  ∀ m, m < n ->
    ¬ ExistsMediatedR2CertificateAtDim
      obs_alignment sigma_alignment I_alignment m

no_smaller_properMediatedR2Certificate_alignment :
  ∀ m, m < n ->
    ¬ ExistsProperMediatedR2CertificateAtDim
      obs_alignment sigma_alignment I_alignment m
```

Then prove:

```lean
exactMediatedR2Dimension_n_alignment :
  ExactMediatedR2Dimension obs_alignment sigma_alignment I_alignment n

exactProperMediatedR2Dimension_n_alignment :
  ExactProperMediatedR2Dimension obs_alignment sigma_alignment I_alignment n
```

The final file should also expose a direct theorem matching the hierarchy
pattern:

```lean
dimensionMinimalProperMediatedR2Certificate_M_alignment :
  DimensionMinimalProperMediatedR2Certificate
    obs_alignment sigma_alignment I_alignment M_alignment
```

## End-To-End Theorem

Expose a final theorem:

```lean
theorem endToEnd_alignment
    {n : Nat} (h : 1 < n) :
    ResidualNonempty_R2 obs_alignment sigma_alignment I_alignment
      ∧ MediatedResidualEmpty obs_alignment sigma_alignment
          I_alignment M_alignment
      ∧ IrreducibleMediator obs_alignment I_alignment M_alignment
      ∧ (∀ m : Nat,
          m < n ->
            ¬ ExistsProperMediatedR2CertificateAtDim
              obs_alignment sigma_alignment I_alignment m)
```

Interpretation:

```text
An observation-only alignment interface leaves a trajectory residual.
The causal mediator closes it.
The exact mediator dimension is n.
Visible marginal interfaces do not recover it.
```

## What This Would Prove

This would prove:

```text
There are abstract alignment states with identical terminal observations but
distinct causal trajectories, and exact finite mediation is required to close
that residual.
```

It would also prove:

```text
For every n >= 2, the minimal causal mediator dimension can be exactly n.
```

## What This Would Not Prove Yet

This would not yet prove:

```text
real LLMs have exactly this carrier;
real transformer internals instantiate this model;
empirical ASLMT logs satisfy the Lean definitions;
alignment is solved for deployed systems.
```

Those require a second bridge layer.

## Bridge To Empirical Work

After the abstract Lean instance is complete, the next target is a bridge
document or Lean-adjacent specification connecting empirical ASLMT terms to
the formal carrier:

```text
empirical context -> PromptClass
terminal output / mask / answer -> TerminalOutput
latent causal selection -> trajectory : Fin n
local/global arbitration -> mediator M
OOD success under hidden trajectory variation -> evidence for residual closure
```

The bridge should remain honest:

```text
Lean proves the structural theorem.
Experiments provide evidence that a given system instantiates the structure.
```
